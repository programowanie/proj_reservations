# Mucha Radosław 
## Dynamiczny System Sprzedaży Biletów

Stworzę program symulujący Dynamiczny System Sprzedaży Biletów, który stosują **przewoźnicy kolejowi** lub autobusowi. 

W zależności od parametrów (dzień powszedni, weekend świąteczny, wakacje) program wylosuje liczbę pasażerów. Następnie odbędzie się losowanie trasy przejazdu. Na tej podstawie program będzie umieszczał w jednym **wagonie**/autobusie pasażerów wraz z przydzielonym miejscem w zależności od ich preferencji (okno/środek/korytarz). Jeżeli zacznie brakować miejsc, program automatycznie doda kolejny pojazd do dyspozycji pasażerów.

Program na bieżąco będzie podawał, ile osób i w jakiej relacji kupiło **bilet** na dane połączenie, a także informacje systemowe na temat frekwencji, relacji i wolnej ilości miejsc. Po ulokowaniu wszystkich pasażerów, program wydrukuje listę pasażerów, relację, zajmowane miejsce, ilość wagonów/autobusów potrzebnych do zrealizowania połączenia.
